# Eval: 换尿布默认小便与统计视图后置

## Prompt

使用 `baby-feeding-summarize` 整理下面记录。注意：用户希望只要是换尿布相关描述，未写明有大便时就按小便统计；统计视图放到文档最后；喂养分析要判断当前状态、问题和优化建议。

## Input

```text
【宝宝护理记录｜6月29日】

02:00 换尿不湿
03:10 喝母乳 60 ml
03:18 拍嗝 5 分钟
06:30 换尿布：大便无
07:00 吸母乳左右各 12 分钟
07:25 奶粉 40 ml
09:30 换纸尿裤，干净，未见大便
10:20 大便黄色，糊状，量中
12:00 换尿不湿：小便无，大便有
12:02 大便黄色，糊状，量中
13:20 奶粉 80 ml
```

## 成功标准

- 02:00、06:30、09:30 都属于换尿布相关描述，且未写明有大便，应按小便统计。
- 12:00 明确写大便有，应计入大便；是否小便只按原文明确尿便线索判断，不能因“换尿不湿”再自动计小便。
- 10:20 和 12:00/12:02 应统计为 2 次大便；12:00 和 12:02 是同一次大便事件和质量描述，不能重复计数。
- 文档主体顺序应先给今日概览、喂养分析、排便排尿分析、睡眠推测、观察优先级、信息缺口，最后才出现 `### 统计视图`。
- `### 喂养分析` 必须包含“当前状态判断”：明确哪些方面较好、哪些是潜在问题，问题要结合证据描述，并给出优化建议。
- `### 统计视图` 中再集中展示趋势速览、参考区间对比、关键数据和喂养时间线。
