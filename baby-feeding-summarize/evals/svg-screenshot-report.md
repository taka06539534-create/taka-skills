# Eval: SVG 与截图 Markdown 报告

## Prompt

使用 `baby-feeding-summarize` 处理一份新生儿喂养记录。用户明确要求生成更直观的可视化图表，并把 SVG 和截图放到最终 Markdown 产物中。

## 成功标准

- 应先把原始记录整理为结构化事件数据，再生成可视化产物。
- 必须生成或指导生成一个 SVG 图表，至少包含：
  - 24 小时时间轴，分轨展示喂养、尿便、睡眠。
  - 瓶喂量柱状图，并标出常见参考区间。
  - 关键指标卡片，例如瓶喂总量、亲喂总时长、明确小便次数、大便次数。
- 最终 Markdown 产物只应保留一个“可视化图表”章节，不应额外创建“截图”章节。
- 如果环境可以生成 PNG 截图，最终 Markdown 应在“可视化图表”章节展示 PNG，并提供 SVG 源文件链接。
- 如果无法生成截图，最终 Markdown 应在“可视化图表”章节展示 SVG，并说明缺少浏览器或 SVG 转 PNG 能力。
- 不应把参考区间写成诊断结论；对比措辞使用“接近参考 / 记录不足 / 需结合日龄观察”等。
- 脚本或流程必须是非交互式，可在 Codex、Claude Code、OpenClaw 等 Agent 平台复用。
