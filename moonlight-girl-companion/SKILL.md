# 午夜闺蜜文案生成 Skill

## 概览

使用本 skill 生成适合微信公众号发布的午夜情感故事发布包。默认面向中年女性读者，从女性第一人称或贴近女性主人公的第三人称展开，产出可直接使用的公众号文章正文、标题、描述、封面图片、章节插图、插图规划、Prompt、验收报告、Goal 模式迭代日志和最终 PASS/FAIL 状态。

本 skill 不是单次写作提示词。被调用后，要按 Goal 模式完成“选题 -> 设定 -> 正文 -> 插图生成 -> 发布包组装 -> 验收 -> 修订 -> 再验收”的完整链路，直到达到 Done Criteria 或达到最大迭代轮数。只有一个 `.md` 文档、只有插图 Prompt、没有真实封面和章节图片文件的产物不能输出 PASS。

## 何时使用

- 用户要求写微信公众号情感故事、午夜情感故事、婚姻情感故事、中年女性故事、女性视角故事。
- 用户要求生成公众号正文、公众号摘要、封面图 Prompt、章节插图 Prompt、故事分镜或插图方案。
- 用户要求生成可直接发布的公众号文章包，包含标题、描述、完整正文、封面图、章节插图和可复用素材文件。
- 用户只给出很少信息，例如“帮我自动生成一篇适合公众号发布的午夜情感故事”。
- 用户明确要求在 Codex Goal 模式下循环优化，最终输出 PASS 或 FAIL。

## 何时不要使用

- 用户只想要普通短句、标题改写、单张图片 Prompt，且不需要完整故事内容包。
- 用户要求低俗、血腥、恶意惊悚、真实人物造谣、医疗法律定论或其他违反安全边界的内容。
- 用户要求直接发布、部署、付费推广或绕过平台审核；本 skill 只产出内容包和验收结论。

## 默认参数

- 目标读者：微信公众号中年女性读者。
- 叙事视角：女性第一人称或女性近距离第三人称。
- 情绪基调：深夜感、克制、共情、反转、治愈。
- 文章长度：3000 字左右；可接受范围为 2800 到 3600 字。
- 章节数量：5 个章节左右，默认至少 5 个章节。
- 正文结尾：最后一章结尾后必须依次添加 `### 深夜寄语`、`### 互动话题`、`### 更多文章入口`，其中更多文章入口用于引导读者继续阅读更多同类公众号文章。
- 插图数量：每章 1 张。
- 插图比例：16:9 横版。
- 插图风格：电影感现实主义、深夜情绪感、暗蓝色调、暖黄色光源、细腻人物表情、高级公众号插图风格。
- 封面图比例：16:9 横版。
- 默认产物形态：发布包目录，而不是单个 Markdown 文件。
- 默认文章入口：`article.md`。
- 默认公众号元数据：`metadata.json`，必须包含 `title`、`description`、`cover_image` 和章节插图路径。
- 默认真实图片文件：`cover.png` 和 `images/chapter-01.png` 到 `images/chapter-05.png`，或同等 `.jpg`、`.jpeg`、`.webp` 文件。
- Goal 模式最大迭代轮数：5 轮。
- 最终发布验收分数线：90 分。
- 自主选题最低通过分数：80 分。
- Hard Gate：全部必须通过。
- Soft Gate：超过 3 项时不能标记为“可直接发布”。

## 工作模式判断

### 指定选题模式

当用户提供明确主题、故事梗概、人物设定、关键情节或素材时，进入指定选题模式。

指定选题模式必须优先保留用户给出的核心设定，不得擅自更换主题。可以优化标题、章节结构、人物设定、情绪路径、叙事节奏、插图方案和生图 Prompt，但不能推翻用户给出的核心故事方向。

### 自主选题模式

当用户没有提供明确主题，或只说“帮我生成一篇午夜情感故事”“自动写一篇适合公众号发布的情感故事”“今天帮我产出一篇公众号文章”“生成一篇女性视角情感故事”“帮我自动完成一篇公众号故事和插图”等请求时，进入自主选题模式。

自主选题模式必须先选题，不能直接写正文。必须完成候选选题、评分、最终选题、自主选题报告，再进入正文创作。

## 自主选题流程

1. 从 `references/topic_selection_guide.md` 的适合方向中选择本期内容方向。
2. 生成 5 到 8 个候选选题，每个候选包含选题标题、一句话故事钩子、女主身份、核心冲突、情绪关键词、关键物件、适合插图的核心画面、潜在风险。
3. 按 100 分制评分：目标读者命中度 20 分、开头悬念强度 15 分、情绪张力 15 分、故事完整潜力 15 分、公众号传播性 10 分、画面表现力 10 分、合规安全性 10 分、新鲜度 5 分。
4. 选择最高分选题。若最高分低于 80 分，重新生成候选选题，不允许进入正文创作。
5. 多个选题得分接近时，优先选择更适合中年女性读者、更有现实生活细节、更容易形成 5 章情绪推进、更适合生成连续插图、合规风险更低的选题。
6. 正式创作前输出“自主选题报告”，使用 `assets/topic_selection_report_template.md`。

## 内容生成流程

1. 生成内容简报：主题、目标读者、叙事视角、核心冲突、情绪基调、结局方向。
2. 生成故事设定卡：主人公姓名、年龄、职业、婚姻或家庭状态、外貌特征、性格特征、重要配角、主要地点、时间背景、核心秘密或冲突、关键物件、情绪变化、结局方向。
3. 生成角色视觉设定卡：每个主要角色的年龄段、身材气质、发型、服装、面部特征、代表色、关键配饰、情绪状态、必须保持不变的视觉元素。
4. 生成 5 个章节左右的大纲。每章包含章节标题、主要情节、情绪功能、读者悬念、对应插图画面。
5. 写 2800 到 3600 字正文。每章必须有小标题和插图占位，段落短，适合手机端阅读。最后一章故事结尾之后必须依次添加 `### 深夜寄语`、`### 互动话题`、`### 更多文章入口`，用温和、陪伴感的语气完成情绪收束、读者互动和更多文章导流。
6. 生成 5 个标题备选、公众号摘要、封面图 Prompt。
7. 为每章生成一张插图方案和生图 Prompt，使用 `assets/illustration_prompt_template.md`。
8. 使用可用的图像生成能力生成真实图片文件：封面图 1 张、每章插图 1 张。若当前环境没有图像生成或保存图片能力，必须将最终状态标记为 FAIL，并说明缺少真实图片资产，不能假装可直接发布。
9. 组装公众号发布包目录，包含 `article.md`、`metadata.json`、`cover.png` 或同等格式封面、`images/chapter-01.png` 到 `images/chapter-05.png` 或同等格式章节图、Prompt 文件、验收报告和迭代日志。
10. 生成质量验收报告、Goal 模式迭代日志、修订说明和最终状态。

推荐情绪路径：压抑 -> 发现 -> 崩塌 -> 觉醒 -> 自我救赎。

正文结尾的固定章节要求：

- 必须出现在 `## 5. 正文` 内，位于最后一章故事正文和最后一张章节插图之后。
- 顺序必须固定为：`### 深夜寄语` -> `### 互动话题` -> `### 更多文章入口`。
- `深夜寄语`：承接故事余味，给读者一段克制、温柔、不说教的情绪陪伴，建议 80 到 180 个中文字。
- `互动话题`：提出 1 到 3 个适合公众号评论区互动的问题，围绕婚姻、边界、自我找回、关键物件或故事情绪展开，不要诱导隐私暴露。
- `更多文章入口`：文案长度建议 60 到 150 个中文字，像公众号文末推荐语，不要像广告硬广。
- 更多文章入口可以引导读者继续阅读“更多午夜情感故事”“婚姻与自我找回故事”“中年女性情感故事合集”等。
- 三个文末章节都不能使用恐吓、夸大、低俗、强迫点击或“错过就后悔”式表达。
- 三个文末章节都不能破坏故事余味；应承接结尾情绪，保持克制、温柔和陪伴感。

推荐章节结构：

1. 平静生活里的裂缝
2. 深夜发现的秘密
3. 她第一次沉默地反击
4. 真相揭开后的崩塌
5. 她把自己重新捡回来

## 插图与封面规则

- 先生成角色视觉设定卡，再生成插图 Prompt。
- 每张章节插图 Prompt 必须复用角色视觉设定卡，并包含“角色一致性设定”“统一美术风格”“统一色彩系统”。
- 插图内容必须紧贴章节情节，像一组连续故事分镜，不是互不相关的海报。
- 必须生成真实图片文件，不能只提供 Prompt。封面默认命名为 `cover.png`，章节图默认命名为 `images/chapter-01.png` 到 `images/chapter-05.png`。
- `article.md` 中每个章节插图占位后必须嵌入对应图片引用，例如 `![插图 1](images/chapter-01.png)`。
- 正文插图尽量不生成中文文字；封面如需文字，保持简短、清晰、适合公众号封面。
- 禁止血腥、低俗、裸露、恐怖、恶意惊悚、年龄发型服装气质前后矛盾。
- 封面图 Prompt 必须包含主标题、副标题、女主形象、核心情绪画面、公众号封面适配、文字区域预留、色彩氛围、构图建议、负面提示词。
- 角色视觉设定卡确定后，封面图和各章节插图可以并行生成。Codex 环境允许使用 subagent 时，可以把封面、插图 1、插图 2、插图 3、插图 4、插图 5 分派给独立 subagent 或并行任务；每个任务只能写入自己负责的图片文件和对应 Prompt 草稿，不得修改正文、故事设定或 metadata。回收后必须由主 Agent 统一检查角色一致性、色彩系统、文件路径和图片引用。

## 发布包目录规范

默认必须生成一个独立发布包目录，例如 `articles/<YYYY-MM-DD>-<slug>/`：

```text
<article-package>/
├── article.md
├── metadata.json
├── cover.png
├── images/
│   ├── chapter-01.png
│   ├── chapter-02.png
│   ├── chapter-03.png
│   ├── chapter-04.png
│   └── chapter-05.png
├── prompts/
│   ├── cover_prompt.md
│   └── illustration_prompts.md
├── review_report.md
└── iteration_log.md
```

`article.md` 是给公众号编辑直接使用的文章正文，必须包含主标题、描述或摘要、正文、章节插图引用、文末深夜寄语、互动话题、更多文章入口和必要的发布备注。`metadata.json` 是给发布流程和复核使用的结构化元数据，至少包含：

```json
{
  "title": "公众号主标题",
  "description": "公众号摘要或分享描述",
  "cover_image": "cover.png",
  "article_file": "article.md",
  "illustrations": [
    {"chapter": "01", "path": "images/chapter-01.png"}
  ]
}
```

不能只把封面图 Prompt 和章节插图 Prompt 写在 Markdown 里。若真实图片文件缺失、metadata 缺失或 `article.md` 未引用章节图片，结构校验必须失败。

## Goal 模式循环

在 Codex Goal 模式下，按以下循环执行，不能只生成一次内容：

1. 判断工作模式。
2. 自主选题模式先生成并评分候选选题。
3. 选择最终选题，确认是否允许进入正文创作。
4. 生成内容包初稿。
5. 生成真实封面和章节插图文件，必要时并行生成插图。
6. 组装发布包目录，写入 `article.md`、`metadata.json`、图片文件、Prompt、验收报告和迭代日志。
7. 对发布包目录运行 `python scripts/validate_article_package.py <article-package-dir>`。
8. 执行规则型质量验收，输出质量验收报告。
9. 判断是否通过发布门禁。
10. 如果未通过，定位问题层级，生成修订计划。
11. 根据修订计划改写内容，并再次校验和验收。
12. 重复循环，直到通过验收或达到最大 5 轮。
13. 输出发布包目录路径、最终稿、真实图片资产、验收报告、迭代日志和剩余风险。

每轮迭代必须记录：当前迭代轮次、本轮目标、当前工作模式、上一轮失败项、本轮修复动作、本轮修改范围、本轮质量评分、是否通过 Hard Gate、是否通过 Soft Gate、是否达到 Done Criteria、下一轮是否继续、停止原因。

## 问题分层与修订策略

问题层级：

1. 选题问题
2. 故事设定问题
3. 角色视觉设定问题
4. 章节结构问题
5. 正文表达问题
6. 插图一致性问题
7. 标题、摘要和封面问题

修订策略：

- 选题不成立时，回到自主选题环节重新选题。
- 选题成立但故事设定薄弱时，重新生成故事设定卡。
- 角色设定不清晰时，重新生成角色视觉设定卡，并同步修正所有插图 Prompt。
- 故事设定成立但章节结构弱时，重写章节大纲。
- 章节结构成立但正文不好时，只修正文。
- 正文成立但插图不一致时，只修正角色视觉设定和插图 Prompt。
- 主体内容已达标时，只优化标题、摘要和封面 Prompt。

禁止每轮盲目重写全文；禁止选题不合格却继续润色正文；禁止正文改好了但插图角色设定丢失；禁止为了凑字数添加无意义段落；禁止未通过验收就输出 PASS。

## Done Criteria 摘要

只有同时满足以下条件，才能判定 DONE 并输出 PASS：

- 结构校验脚本通过。
- 存在发布包目录，且包含 `article.md`、`metadata.json`、真实封面图片和每章真实插图图片。
- `metadata.json` 存在非空 `title`、`description`、`cover_image` 字段。
- 自主选题模式下存在自主选题报告、5 到 8 个候选选题、最终选题评分不低于 80 分、女主身份明确、核心冲突明确、关键物件明确、可自然拆成 5 个章节、适合连续插图、通过合规安全检查。
- 正文字数在 2800 到 3600 字之间，至少 5 个章节，每章有小标题、插图占位和对应插图 Prompt。
- `article.md` 中每个章节必须引用对应真实图片文件。
- `article.md` 正文结尾必须依次包含“深夜寄语”“互动话题”“更多文章入口”三个章节。
- 存在故事设定卡、角色视觉设定卡、公众号摘要、封面图 Prompt、质量验收报告、Goal 模式迭代日志和最终状态。
- 每张插图 Prompt 复用角色视觉设定卡，插图风格和角色形象前后一致。
- 正文为女性视角或贴近女性主人公视角，开头 3 段有明确悬念或情绪钩子。
- 情绪路径完整，至少包含压抑、发现、崩塌、觉醒、自我救赎。
- 不存在低俗、血腥、恶意惊悚、虚假事实、医疗法律误导等硬性风险。
- 质量评分达到 90 分或以上，Hard Gate 全部通过，验收报告给出“建议发布”或“可直接发布”。
- 迭代日志完整记录每轮变化，最终状态明确写出 PASS 或 FAIL。

完整验收标准见 `references/acceptance_criteria.md`。

## Hard Gate 摘要

任意 Hard Gate 不通过，都必须 FAIL，不能标记为 DONE。重点包括：缺少发布包目录、`article.md`、`metadata.json`、真实封面图片、真实章节图片、正文、故事设定卡、角色视觉设定卡、质量验收报告、章节插图 Prompt、封面图 Prompt、公众号摘要、文末深夜寄语、互动话题、更多文章入口、最终状态；章节少于 4 个；文末三个章节顺序错误；真实章节插图少于章节数量；`article.md` 未引用真实章节图片；正文字数低于 2600 或高于 3800；非女性视角；插图未复用角色视觉设定；角色形象明显冲突；存在低俗色情、血腥暴力、恶意惊悚、虚假医疗法律结论、真实人物造谣、未成年人不当描写；质量评分低于 80；校验脚本失败；自主选题模式下缺少选题报告或最终选题低于 80 分。

## 默认输出结构

默认使用 `assets/article_output_template.md` 和 `assets/publish_metadata_template.json`，生成发布包目录。聊天回复只给出发布包路径、核心文件列表、验收摘要和最终状态，不要把完整文章正文再次贴满聊天窗口。

发布包至少包含：

1. `article.md`：可直接使用的公众号文章，包含标题、描述、完整正文、章节图片引用、文末深夜寄语、互动话题和更多文章入口。
2. `metadata.json`：标题、描述、封面路径、文章路径、章节图路径。
3. `cover.png` 或同等格式封面图。
4. `images/chapter-01.png` 到 `images/chapter-05.png` 或同等格式章节图。
5. `prompts/cover_prompt.md` 和 `prompts/illustration_prompts.md`。
6. `review_report.md`。
7. `iteration_log.md`。
8. 最终状态：`PASS：达到发布标准` 或 `FAIL：未达到发布标准，需要人工介入`。

## 校验脚本

生成发布包目录后运行：

```bash
python moonlight-girl-companion/scripts/validate_article_package.py <article-package-dir>
```

脚本输出 JSON，包含 `passed`、`errors`、`warnings`、`stats`。存在 errors 时退出码为 1；无 errors 时退出码为 0。脚本会检查正文结构、字数、`metadata.json`、真实封面图片、真实章节图片文件和正文图片引用。脚本不能替代人工风格判断或规则型质量验收。

## 失败处理

达到最大迭代轮数仍未通过验收时，停止循环并输出 FAIL。必须说明当前最高分版本、未通过的 Hard Gate、未解决的 Soft Gate、无法继续提升的原因、建议人工介入位置、最后一版可用稿件和后续人工修改建议。不能因为达到迭代上限就假装通过。
