# Goal 模式循环协议

## 目标

在 Codex Goal 模式下，循环执行 moonlight-girl-companion skill，通过多轮选题、生成、验收和修订，产出达到公众号发布标准的午夜情感故事内容包。

## 每轮循环步骤

1. 读取用户需求。
2. 判断工作模式：指定选题模式或自主选题模式。
3. 如果是自主选题模式，先执行选题流程。
4. 读取上一轮输出和验收报告。
5. 判断上一轮是否通过 Done Criteria。
6. 如果已经通过，停止循环并输出 PASS。
7. 如果未通过，提取失败项。
8. 判断失败项所属层级。
9. 按优先级制定修订计划。
10. 执行修订。
11. 运行校验脚本。
12. 输出新的质量验收报告。
13. 更新迭代日志。
14. 判断是否继续下一轮。

## 修订优先级

第一优先级：合规安全问题
第二优先级：选题问题
第三优先级：结构缺失问题
第四优先级：字数和章节问题
第五优先级：女性视角和人物逻辑问题
第六优先级：角色视觉设定问题
第七优先级：插图一致性问题
第八优先级：情绪推进和故事节奏问题
第九优先级：标题、摘要、语言润色问题

## 停止条件

满足以下任意条件时停止：

1. Done Criteria 全部满足，输出 PASS。
2. 达到最大迭代轮数，输出 FAIL。
3. 出现无法自动修复的问题，输出 FAIL 并请求人工介入。
4. 用户明确要求停止。

## 禁止行为

1. 不允许未通过验收就输出 PASS。
2. 不允许只给建议不执行修订。
3. 不允许每轮重复生成全新故事，除非结构严重失败或选题不成立。
4. 不允许修订后丢失用户原始核心要求。
5. 不允许修订后导致角色视觉设定变化。
6. 不允许忽略上一轮验收报告。
7. 不允许为了凑字数添加无意义段落。
8. 不允许为了情绪刺激制造低俗或猎奇内容。
9. 不允许选题不合格时继续润色正文。
10. 不允许自主选题模式下跳过候选选题评分。
