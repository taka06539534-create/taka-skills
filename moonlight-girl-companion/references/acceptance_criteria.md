# 最终验收标准

## 必须通过项

1. 有完整内容简报。
2. 有完整故事设定卡。
3. 有完整角色视觉设定卡。
4. 有 5 个左右章节。
5. 有 2800 到 3600 字正文。
6. 有发布包目录，不是只有单个 Markdown 文件。
7. 发布包目录中有 `article.md` 或 `index.md`。
8. 发布包目录中有 `metadata.json`。
9. `metadata.json` 有非空 `title`、`description`、`cover_image`。
10. 有真实封面图片文件，默认 `cover.png`，可使用 `.jpg`、`.jpeg`、`.webp`。
11. 有不少于章节数量的真实章节插图文件，默认放在 `images/chapter-01.png` 到 `images/chapter-05.png`。
12. `article.md` 正文中引用了每个章节对应的真实插图文件。
13. `article.md` 正文结尾依次有“深夜寄语”“互动话题”“更多文章入口”，用于完成情绪陪伴、读者互动和进入其他更多文章。
14. 每个章节都有插图占位。
15. 每个章节都有插图 Prompt。
16. 每张插图 Prompt 都有角色一致性设定。
17. 每张插图 Prompt 都有统一美术风格。
18. 每张插图 Prompt 都有统一色彩系统。
19. 有封面图 Prompt。
20. 有公众号摘要。
21. 有质量验收报告。
22. 有 Goal 模式迭代日志。
23. 有最终 PASS 或 FAIL 状态。
24. 质量评分不低于 90 分。
25. Hard Gate 全部通过。
26. 如果是自主选题模式，有自主选题报告。
27. 如果是自主选题模式，候选选题不少于 5 个。
28. 如果是自主选题模式，最终选题评分不低于 80 分。

## Goal 模式 Done Criteria

只有同时满足以下条件，才可以判定为 DONE：

1. 结构校验脚本通过。
2. 存在发布包目录，且目录中有 `article.md` 或 `index.md`。
3. 存在 `metadata.json`，且有非空 `title`、`description`、`cover_image`。
4. 存在真实可打开的封面图片文件。
5. 存在不少于章节数量的真实章节插图图片文件。
6. 正文中每章都引用了对应真实章节插图。
7. 正文结尾依次有“深夜寄语”“互动话题”“更多文章入口”，且不破坏故事余味。
8. 如果是自主选题模式，必须存在自主选题报告。
9. 如果是自主选题模式，必须有 5 到 8 个候选选题。
10. 如果是自主选题模式，最终选题评分必须不低于 80 分。
11. 如果是自主选题模式，最终选题必须有明确女主身份。
12. 如果是自主选题模式，最终选题必须有明确核心冲突。
13. 如果是自主选题模式，最终选题必须有关键物件。
14. 如果是自主选题模式，最终选题必须能自然拆成 5 个章节。
15. 如果是自主选题模式，最终选题必须适合生成连续插图。
16. 如果是自主选题模式，最终选题必须通过合规安全检查。
17. 正文中文字数在 2800 到 3600 字之间。
18. 正文包含 5 个章节左右，默认至少 5 个章节。
19. 每个章节都有明确小标题。
20. 每个章节都有插图占位。
21. 每个章节都有对应插图 Prompt。
22. 存在故事设定卡。
23. 存在角色视觉设定卡。
24. 每张插图 Prompt 都复用了角色视觉设定卡。
25. 插图风格前后一致。
26. 女主、男主和重要配角形象前后一致。
27. 正文为女性视角或贴近女性主人公视角。
28. 开头 3 段具备明确悬念或情绪钩子。
29. 故事情绪路径完整，至少包含压抑、发现、崩塌、觉醒、自我救赎。
30. 结尾有余味，不能突然中断。
31. 不存在低俗、血腥、恶意惊悚、虚假事实、医疗法律误导等硬性风险。
32. 质量评分达到 90 分或以上。
33. Hard Gate 全部通过。
34. 质量验收报告给出“建议发布”或“可直接发布”。
35. 迭代日志完整记录每轮变化。
36. 最终状态明确写出 PASS 或 FAIL。

## Hard Gate 硬性门禁

以下任意一项不通过，都必须判定为 FAIL，不能标记为 DONE：

1. 缺少正文。
2. 缺少故事设定卡。
3. 缺少角色视觉设定卡。
4. 缺少质量验收报告。
5. 缺少章节插图 Prompt。
6. 缺少封面图 Prompt。
7. 缺少公众号摘要。
8. 缺少最终状态。
9. 章节数量少于 4 个。
10. 缺少发布包目录，或只有一个 `.md` 文件。
11. 缺少 `metadata.json`。
12. `metadata.json` 缺少标题、描述或封面图路径。
13. 缺少真实封面图片文件。
14. 真实章节插图数量少于章节数量。
15. `article.md` 没有引用章节插图文件。
16. 正文结尾缺少“深夜寄语”“互动话题”或“更多文章入口”。
17. 正文结尾的“深夜寄语”“互动话题”“更多文章入口”顺序错误。
18. 插图数量少于章节数量。
19. 正文字数低于 2600 字或高于 3800 字。
20. 文章不是女性视角，也没有贴近女性主人公。
21. 插图 Prompt 没有复用角色视觉设定。
22. 同一角色在插图中出现明显形象冲突。
23. 存在低俗色情内容。
24. 存在血腥暴力细节。
25. 存在恶意惊悚或恐吓式标题。
26. 存在虚假医疗建议。
27. 存在虚假法律结论。
28. 存在真实人物造谣。
29. 存在对未成年人的不当描写。
30. 质量评分低于 80 分。
31. 校验脚本执行失败。
32. 自主选题模式下缺少自主选题报告。
33. 自主选题模式下候选选题少于 5 个。
34. 自主选题模式下最终选题评分低于 80 分。
35. 自主选题模式下最终选题没有明确女主身份。
36. 自主选题模式下最终选题没有核心冲突。
37. 自主选题模式下最终选题没有关键物件。
38. 自主选题模式下最终选题无法支撑 5 个章节。
39. 自主选题模式下最终选题不适合生成连续插图。
40. 自主选题模式下最终选题存在明显低俗、血腥、恶意惊悚或造谣风险。

## Soft Gate 软性门禁

以下问题不会直接阻止输出，但必须进入修订建议：

1. 标题吸引力不足。
2. 开头钩子较弱。
3. 情绪推进不够明显。
4. 中间章节节奏拖沓。
5. 结尾偏鸡汤。
6. 部分段落过长。
7. 插图画面不够贴合情节。
8. 生图 Prompt 不够具体。
9. 封面图文字过多。
10. 公众号摘要不够吸引人。
11. 候选选题新鲜度不足。
12. 女主情绪变化不够自然。
13. 配角动机不够清晰。
14. 关键物件没有贯穿全文。
15. 深夜寄语过度说教、互动话题过度索取隐私，或更多文章入口过硬、过像广告、与故事情绪脱节。

Soft Gate 问题超过 3 项时，不能标记为“可直接发布”，只能标记为“修改后发布”或继续迭代。

## 建议通过项

1. 标题有点击欲望，但不低俗。
2. 开头 3 段能建立悬念。
3. 每章都有明确情绪转折。
4. 结尾有余味。
5. 插图像一组连续故事分镜。
6. 公众号手机端阅读体验好。
7. 段落长度适中。
8. 没有明显重复表达。
9. 没有空洞说教。
10. 没有人物行为突兀问题。
11. 关键物件贯穿全文。
12. 封面图有明显情绪记忆点。
13. 真实图片文件命名清晰，能直接上传或转存到公众号后台。
14. `article.md` 中的图片引用顺序与章节顺序一致。
15. 文末深夜寄语、互动话题和更多文章入口自然承接故事余味，能引导读者评论与继续阅读同类文章。

## 最终输出要求

最终必须明确写出：

最终状态：PASS 或 FAIL

如果 PASS，说明：

* 质量评分
* 通过的 Hard Gate
* 剩余 Soft Gate
* 是否建议发布
* 发布包目录
* 封面图片路径
* 文章文件路径
* 章节图片路径

如果 FAIL，说明：

* 失败原因
* 未通过的门禁
* 已完成的修订
* 仍需人工介入的内容
