# 插图一致性指南

1. 每篇故事必须先生成角色视觉设定卡，再生成插图 Prompt。
2. 每张插图 Prompt 必须复用角色视觉设定卡。
3. 不允许女主前一张是长发，后一张突然变短发，除非剧情明确写了剪发。
4. 不允许服装频繁变化，除非剧情发生明显时间跳跃。
5. 不允许年龄感变化过大。
6. 不允许画风从写实突然变成漫画、古风、赛博朋克等。
7. 所有图都要保持同一色彩系统。
8. 插图应该服务情节，而不是单纯好看。
9. 插图要有连续镜头感，像一组故事分镜。
10. 封面可以更有设计感，正文插图更偏剧情场景。
11. 每张插图都必须明确人物、场景、情绪、光线、色彩、构图。
12. 每张插图都必须包含负面提示词。
13. 正文插图尽量不生成中文文字。
14. 如需封面文字，必须保持简短、清晰、适合公众号封面。
15. Goal 模式下，每一轮修订都必须检查角色视觉一致性。
16. 角色视觉一致性不通过时，不能输出 PASS。
17. 每张插图 Prompt 必须包含“角色一致性设定”字段。
18. 每张插图 Prompt 必须包含“统一美术风格”字段。
19. 每张插图 Prompt 必须包含“统一色彩系统”字段。
20. 插图的光线和色彩应与故事情绪推进一致。
21. 插图应从封面到第 5 章形成连续的视觉叙事。
22. 不允许每张图都只是孤独女人背影，必须结合具体情节变化。
23. 最终发布包必须包含真实封面图片文件和真实章节插图文件，不能只有 Prompt。
24. 默认封面文件为 `cover.png`，默认章节图为 `images/chapter-01.png` 到 `images/chapter-05.png`，也可使用 `.jpg`、`.jpeg`、`.webp`。
25. `article.md` 中每个章节插图占位后必须引用对应图片文件。
26. 并行生成图片时，必须先锁定角色视觉设定卡；每个并行任务只能生成自己负责的图片，不得改动角色设定。
27. 使用 subagent 并行生成图片后，主 Agent 必须统一检查女主、男主、配角、服装、发型、关键配饰、色彩系统和镜头连续性。
28. 任一真实图片缺失、打不开或与角色视觉设定冲突时，不能输出 PASS。
